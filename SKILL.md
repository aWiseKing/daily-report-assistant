# 日报助手

## 目标

在指定目录中管理当日的日报文件：

- 构建：创建当日文件 `日报_YYYY_MM_DD.md`
- 添加：向当日文件追加日报信息
- 删除/调整：删除或修改当日文件中的对应信息

## 工作目录与文件命名

- 日报目录：默认使用"G:\file\工作汇报\周报"目录；如用户明确“规定目录”，以用户给出的目录为准
- 文件名：`日报_YYYY_MM_DD.md`（月、日使用两位数补零）
- 示例：`日报_2026_03_11.md`

## 交互约定（建议）

为保证后续“删除/调整对应信息”可定位，所有新增条目都按“带编号条目”的格式写入。

### 条目格式

- 在文件中以列表项写入：
  - `- [#<编号>] <内容>`
- 编号从 `1` 开始递增，按文件内已有最大编号 + 1 生成

### 指令示例

用户可能用自然语言表达，按语义识别即可：

- 构建：`构建今天日报` / `生成日报` / `创建日报文件`
- 添加：`添加：完成了登录页重构` / `把这条也记到日报：修复支付回调`
- 删除：`删除 #3` / `把第三条删掉`
- 调整：`把 #2 改成：完成接口联调并补充单测`

## 执行规则

### 1) 构建日报（创建当日文件）

当接到“构建日报”类指令：

1. 计算今天日期（以本地日期为准）
2. 生成当日文件路径：`<日报目录>\日报_YYYY_MM_DD.md`
3. 若文件不存在：创建文件并写入基础结构（可为空也可包含标题）
4. 若文件已存在：不覆盖，仅返回“已存在”的结果

建议的基础结构（可按用户习惯调整）：

```markdown
# 日报（YYYY-MM-DD）

## 今日完成

## 问题与风险

## 明日计划
```

### 2) 添加日报信息（追加到当日文件）

当接到“添加日报信息”类指令：

1. 定位当日文件；若不存在则先按“构建日报”规则创建
2. 判断用户给的信息属于哪个板块（若用户未指定，默认追加到“今日完成”）
3. 在目标板块末尾追加一条带编号的列表项：`- [#N] <内容>`

板块识别优先级（从高到低）：

- 用户明确指定板块关键词（如“今日完成/明日计划/问题/风险”）
- 句式意图（如“明天/下周/计划”更偏“明日计划”）
- 默认“今日完成”

### 3) 删除或调整日报信息（删除/修改对应条目）

当接到“删除/调整”类指令：

1. 定位当日文件；若不存在则返回“当日文件不存在，无法删除/调整”
2. 优先用“编号”定位：
   - 删除：删除匹配 `- [#<编号>]` 的整行
   - 调整：将该行 `<内容>` 替换为新内容，保留编号不变
3. 若用户未提供编号，则用“精确文本匹配”定位：
   - 删除：删除完全匹配用户给出的原句的条目行
   - 调整：将匹配到的条目内容替换为新内容
4. 若存在多处匹配：选择最接近“今日完成”板块的匹配；仍不唯一则返回候选列表让用户选择

## 边界与安全

- 不创建未来日期的日报文件，除非用户明确要求某日期
- 只在“日报目录”下读写，避免越权写入其他路径
- 任何改动都在写入前先读取原文件，基于最新内容再做替换/追加
